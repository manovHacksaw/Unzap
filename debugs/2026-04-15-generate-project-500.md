# Debug: /api/generate-project → 500 on Vercel

**Date:** 2026-04-15  
**Endpoint:** `POST /api/generate-project`  
**Symptom:** Works locally, 500 in production on every request.

---

## What the route does

Reads template files from `templates/nextjs/` using `fs.readFileSync`, generates a Next.js project scaffold, zips it, and returns it for download.

---

## Issues encountered (in order)

### 1. Assumed file tracing was the problem — it wasn't

**Hypothesis:** Vercel's static file tracer (`@vercel/nft`) wasn't picking up the `templates/` directory because `fs.readFileSync` was called with a dynamic path (`path.join(process.cwd(), "templates", ...)`).

**Fix attempted:** Added `outputFileTracingIncludes` to `next.config.ts`.

**Result:** Didn't help. The local `.nft.json` already had 26 of 27 template files traced. `outputFileTracingIncludes` is for Next.js standalone output mode and Vercel doesn't fully respect it.

---

### 2. `includeFiles` in `vercel.json` silently drops dotfiles

**Discovery:** Added `includeFiles: "templates/**"` to `vercel.json` for the route. This bundled most files but silently excluded `.env.example` because it starts with `.` — glob implementations (including Vercel's) exclude dotfiles by default without any warning.

**Error:** `ENOENT: no such file or directory, open '/var/task/templates/nextjs/.env.example'`

**Fix:** Renamed `templates/nextjs/.env.example` → `templates/nextjs/env.example` (no leading dot). Updated `projectGenerator.ts` to read from `env.example` but still write it as `.env.example` in the zip output.

---

### 3. `includeFiles` also silently drops `.md` files

**Discovery:** After fixing `.env.example`, the next error was on `README.md`. The `templates/**` glob also excluded markdown files from the bundle — same silent behavior, different file type.

**Error:** `ENOENT: no such file or directory, open '/var/task/templates/nextjs/README.md'`

**Fix attempted:** Changed `includeFiles` to an array `["templates/**", "templates/**/.*"]` to explicitly include dotfiles.

**Result:** Vercel schema validation rejected it — `includeFiles` only accepts a string, not an array.

---

### 4. Switched strategy: embed templates in the JS bundle at build time

**Root insight:** `includeFiles` is unreliable for arbitrary file types. The real fix is to eliminate `fs` at runtime entirely.

**Fix:**
- Created `scripts/bundle-templates.mjs` — reads all files under `templates/` recursively and generates `lib/codegen/templateContents.ts`, a TypeScript module that exports every template as a string literal.
- Updated `lib/codegen/inject.ts` to import from `templateContents.ts` instead of calling `fs.readFileSync`.
- Updated `vercel.json` build command to run the script before `next build`:
  ```
  node scripts/bundle-templates.mjs && next build
  ```

---

### 5. Key mismatch between bundle and lookup

**Discovery:** The bundle script rooted at `templates/nextjs/`, generating keys like `"postcss.config.mjs"`. But `readTemplate("nextjs", "postcss.config.mjs")` looked up `"nextjs/postcss.config.mjs"` — no match, every lookup returned undefined.

**Error:** `Template not found: nextjs/postcss.config.mjs. Run scripts/bundle-templates.mjs to regenerate.`

**Fix:** Changed the bundle script root from `templates/nextjs/` to `templates/`, so keys include the `nextjs/` prefix and match the lookup paths exactly.

---

## Final state

- `scripts/bundle-templates.mjs` — runs at build time, generates `templateContents.ts`
- `lib/codegen/templateContents.ts` — auto-generated, committed to repo, also regenerated by Vercel on every deploy
- `lib/codegen/inject.ts` — reads from `TEMPLATES` map, no `fs` usage
- `vercel.json` — build command runs the script, no `includeFiles` needed

**Rule of thumb:** If a Next.js API route needs to read non-JS files at runtime on Vercel, embed them in the JS bundle at build time. Don't rely on `includeFiles`, `outputFileTracingIncludes`, or dynamic filesystem reads.

---

## When to add new templates

Run locally after any change to `templates/`:
```bash
node scripts/bundle-templates.mjs
git add lib/codegen/templateContents.ts
```

Or just push — Vercel regenerates it automatically during build.
